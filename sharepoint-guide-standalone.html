Managing Permissions

Step-by-step instructions for reviewing and removing access at the site, folder, and file level.

🌐

Site-level permissions apply broadly — removing a user here revokes their access to all documents within the entire site. Use this when you want to fully off-board someone from a SharePoint site.

1
Go to the SharePoint site
Navigate ⌄

Open the SharePoint site where you want to manage permissions. You can find it via your SharePoint home, a bookmark, or through Microsoft 365.

💡 Make sure you have Site Owner or admin permissions — otherwise the Settings menu won't show permission options.
2
Click ⚙ Settings (top right)
Click ⌄

The gear icon (⚙) is in the top-right corner of the SharePoint interface, next to your profile picture. Click it to open the Settings panel.

3
Select "Site permissions"
Click ⌄

In the Settings dropdown, find and click Site permissions. This opens the Permissions panel on the right side of the screen.

📸 Settings menu — click "Site permissions"
4
Select the user and remove them
Action ⌄

Once you've clicked on permissions, you'll be presented with the options below. Select the user you would like to edit and remove them.

📸 Permissions panel — showing site owner, member and visitor groups
⚠️ Removing a user from the site level immediately revokes their access to all content on this site — documents, lists, pages, and more.

✅ Result

That user will no longer have access to any documents within the site.

This is different because files and folders can have unique permissions that differ from the parent site. A few extra steps are needed to reach the Advanced Settings page where individual group memberships can be managed.

1
Open the document library
Navigate ⌄

From the SharePoint site's left navigation, click on Documents (or the relevant document library) to open it.

2
Select the file or folder
Click ⌄

Click the checkbox next to the file or folder name to select it. Don't open it — just select it so the row highlights.
